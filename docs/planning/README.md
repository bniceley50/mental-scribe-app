# Planning & Shipping Documentation

This directory contains all documentation related to project planning, release management, and shipping processes.

## Purpose

These documents have been consolidated here to reduce root directory clutter while maintaining accessibility for team reference.

## Contents

### Shipping Checklists
- **SHIP_CHECKLIST.md** - Main shipping checklist
- **WINDOWS_SHIP_CHECKLIST.md** - Windows-specific shipping checklist
- **GO_NO_GO_CHECKLIST.md** - Decision criteria for releases
- **DEPLOY_READY.md** - Deployment readiness verification
- **POST_DEPLOY_QUICK_CHECK.md** - Post-deployment validation steps

### Release Planning
- **SHIP_PLAN.md** - Overall shipping strategy
- **COMPLETE_SHIP_PACKAGE.md** - Complete shipping package documentation
- **PRESS_GO.md** - Final go-live procedures
- **QUICK_SHIP_CARD_2025-10-21.md** - Quick reference card

### Progress Tracking
- **FEATURE_PROGRESS.md** - Feature implementation progress
- **ALL_FEATURES_COMPLETE.md** - Completed features reference
- **SHIP_SUMMARY_PAGINATION_V1.md** - Version-specific shipping summary

### PR & Release Management
- **PR_BODY.md** - Pull request templates
- **PR_REVIEW_CHECKLIST.md** - PR review guidelines
- **RELEASE_NOTES_TEMPLATES.md** - Templates for release notes
- **RELEASE_v1.3.0.md** - Version 1.3.0 release notes

## Usage

These documents are reference materials for the development and release process. Refer to them when:
- Preparing for a release
- Conducting code reviews
- Planning feature work
- Documenting releases

## Organization Philosophy

We maintain these planning docs separately from technical documentation (in `/docs`) and keep only essential files at the repository root:
- README.md - Project overview
- CONTRIBUTING.md - Contribution guidelines
- CHANGELOG.md - Version history
- SECURITY.md - Security policies
